### 1、快速上手，创建`vuepress`

1. 创建并进入一个新目录

   ```bash
   mkdir vuepress-starter && cd vuepress-starter
   ```

2. 使用你喜欢的包管理器进行初始化

   ```bash
   yarn init # npm init
   ```

3. 将 `VuePress `安装为本地依赖

   ```bash
   yarn add -D vuepress # npm install -D vuepress
   ```

4. 创建你的第一篇文档

   ```bash
   mkdir docs && echo '# Hello VuePress' > docs/README.md
   ```

5. 在 `package.json` 中添加一些 scripts

   ```json
   {
     "scripts": {
       "docs:dev": "vuepress dev docs",
       "docs:build": "vuepress build docs"
     }
   }
   ```

6. 在本地启动服务器

   ```bash
   yarn docs:dev # npm run docs:dev
   ```

   `VuePress` 会在 http://localhost:8080 启动一个热重载的开发服务器。

### 2、把`vuepress`放到`GitHub`

1. 在你的`GitHub`创建`vuepress-starter`仓库。

2. 将你本地的`vuepress`提交推送到你的git仓库

   ```bash
   git init
   git add
   git commit -m "first commit/xxxx" 
   git branch -M main #创建main分支
   git remote add origin git@github.com:dongdongxiao-git/cs.git #设置仓库，这里需要设置你们自己的仓库
   git push -u origin main
   ```

3. 如若`GitHub`提交失败，请检查是否给自己电脑设置`GitHub`秘钥权限。（`GitHub`秘钥配置，自行百度）

### 3、博客部署

> `Vercel`可以与`GitHub`直接关联，当你推送内容至`GitHub`之后，`Vercel`将会自动将博客重新编译和发布。

整个配置过程比较简单

	1. 使用访问`Vercel`页面，直接使用`GitHub`关联登录。
	2. 新建项目，并在下列找到你想进行导入到仓库

![image-20211218155854066](https://gitee.com/xdd_1_1159962709/picBed/raw/master/img/20211218160050.png)

![image-20211218160024763](https://gitee.com/xdd_1_1159962709/picBed/raw/master/img/20211222134522.png)

	3. 点击导入按钮，进行构建参数设置

![image-20211218160216579](https://gitee.com/xdd_1_1159962709/picBed/raw/master/img/20211218160216.png)

	4. 点击`Deploy`依赖按钮，等待其完成构建部署。部署完成后就可以根据其提供的链接访问到vuepress项目了。
	5. 针对其提供的默认链接地址不喜欢，可以在设置`Domains`里进行调整

![image-20211218160602882](https://gitee.com/xdd_1_1159962709/picBed/raw/master/img/20211218160602.png)

有两点需要注意：

​	1. 构建时需要设置一下两个参数，此参数和你`vuepress`项目中`package.json` 的 scripts设置的参数是对应上的。

```
yarn docs:build
docs/.vuepress/dist
```

![image-20211218155223669](https://gitee.com/xdd_1_1159962709/picBed/raw/master/img/20211218155229.png)

	2. 域名配置需要以`.vercel.app`结尾，否则可能需要配置`DNS`等内容，比较麻烦。
